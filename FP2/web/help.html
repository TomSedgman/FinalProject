Logging in
If you have an account, please enter your username and press the ‘Login’ button. Then select from the drop-down menu the project you wish to view and click the ‘View Project’ button.
Please note: if you are a new user then your project will have the name ‘PlaceHolder’.
If you do not have a user account then please click on the ‘Register a new account’ button.


Creating a new account
Please choose a Username. Enter your email address, forename and surname.
Press ‘Index’ to cancel and go back to the Login page, or press ‘Save’ to create a new account.
Creating a new account will create a blank project.


Visualising your data
On first creating a new project, the map will show a blue screen with no markers and the chart object will be blank.
To set up your new project go to the ‘Settings’ button on the sidebar. If you wish to log out and view a different project click ‘Logout’.
If you wish to upload a new file click on the ‘Browse’ button (at the top-right of the screen. Actual wording of the button may vary based on your browser). A pop-up window will appear. Please select the file you wish to upload from your computer. NOTE: Please see file upload guidelines for formatting restrictions.
If you wish to view data then please click on a marker on the map. Select a variable from the drop-down menu. A chart will load displaying the selected data. If you wish to view another variable, please click on the same/a different marker on the map and select another variable.
If you try to load more than two different types of variable a warning message will appear inviting you to either clear the chart via the ‘Clear Chart’ button, or to select variables of the same type as those already chosen.
To zoom in on a specific range of data on the chart, two sliders are provided at the bottom of the screen for restricting the date range. Clicking and dragging either of these will refine the range of data points shown on the chart. To reset the chart click on ‘Clear Chart’.


Setting up your project
From the data visualisation page, click on ‘Settings’. This will navigate you to the Summary page.

Firstly, click ‘Edit project information’. In the ‘Source Variable’ field, enter the column number in which the node identifier value is stored. For more information see the guidelines for file upload.

Enter the project name and any notes about your project in the relevant fields. Click ‘Save’ to return to the summary page.

From there, click ‘Create a New Node Type’. Enter the name of the new node type and click ‘Save’ to return to the summary page.

Now click ‘Create a New Data Definition’. From the drop-down menu, select the node type you have just created. Add a variable name that represents the new data definition (e.g. soil temperature). Enter the unit (e.g. degrees Celsius). Enter the maximum and minimum values expected from this record (e.g. -20 to +40 degrees Celsius). Enter any notes you may wish to store about this data definition (e.g. ‘needs calibrating every 6 weeks’). Choose the data type from the drop-down list (e.g. temperature). In the event of having multiple different variants of the same data types with different (e.g. soil temperature, air temperature, water temperature) they should all be stored as the same type. Enter the variable position ID. This represents where in the incoming data stream this data definition will be found. Please note: the time stamp should always be set as 0. Click ‘Save’ to return to the summary page.

Finally, click ‘Create a New Node’. From the drop-down menu, select the node type you created earlier. Enter the identifier that this node type broadcasts. If the node is a static source then enter the GPS latitude and longitude and ensure that the ‘Is Static?’ option is ticked. If not, ensure ‘Is Static?’ is not ticked and leave the GPS coordinates empty. Click ‘Save’ to return to the summary page.

This data can be edited or deleted at any time by clicking the ‘Edit’ or ‘Destroy’ buttons beside each line.



File upload guidelines
There are a number of restrictions that apply when uploading data:
  • Files must be in .CSV format.
  • The timestamp of the data must be the first column in the text.
  • There must be a field in the data which identifies the source of that record: i.e. If you have three nodes, 1, 2 and 3, then one column in the data must be labeled 1, 2 or 3. Without this your data will not be visible on the graph.


Glossary
Data definition: The meta-data that tells the program how to handle each value in the incoming data-set. Setting up the data definition allows the program to interpret the array of data, including the variable name, the type of parameter, the unit of measurement and the range of acceptable results (max. and min.).

Data type: The restricted list of possible data types.

Marker: A Google Maps icon displaying the location of a node.

Node: The location of a fixed monitoring installation, represented on the map by a marker

Node type: The abstract representation of many nodes sharing the same characteristics.

Variable: The results of one instrument gathered from a node.

Variable position ID: Where in the incoming data stream this data definition will be found. Please note: time stamp should always be set as 0.
